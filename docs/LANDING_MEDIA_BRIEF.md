# INKMADE — Медиа-бриф (весь сайт)

Полный список визуала по всему продукту: что снять/сгенерировать, в каких размерах,
в каком стиле. По каждому пункту — рекомендация **📷 снимать** или **🎨 генерировать (AI)**,
актуальная пропорция (синхронизирована с компонентами) и где это появляется.

> Файл исторически называется `LANDING_*`, но покрывает **весь сайт** (лендинг,
> каталог, товар, конструктор, аккаунт, бренд/мета). На него ссылаются код и заметки —
> имя не меняем.

---

## 🎨 Общий стиль (дизайн-система §2) — единый для всех ассетов

- **Палитра:** бордо `#7A1F28`, крем `#EFE0C1`, графит-чёрный `#111`, тёплый белый `#FBF8F2`.
- **Свет:** мягкий, рассеянный, без жёстких теней. Много воздуха.
- **Эстетика:** стрит-минимализм, премиум, «дорого, но не пафосно».
- **Фоны предметки:** нейтральный крем/светло-серый ИЛИ тёмный бордо/чёрный для драмы.
- **Изделия:** реальные, с реальной печатью. Никаких стоков с чужими логотипами.
- **AI-генерация:** допустима для фонов, атмосферы, OG, иконок-знаков. **НЕ** для фото
  конкретных товаров (это должно быть честное фото реального изделия) и **НЕ** для лиц/UGC,
  выдаваемых за реальных клиентов.
- Перед загрузкой сжимать (TinyPNG / Squoosh).

**Как подключить готовый файл** (для слотов из `app/config/media.ts`):
1. Положить файл в `public/<src>`. 2. Снять `pending: true` у слота. 3. Готово.

# 🟢 БЛОК 3 — Каталог и товар (ОСНОВНОЙ объём, через админку)

Это контент, который ты грузишь в админке для каждого товара. Качество этих фото = качество
всего магазина. Источник: Supabase Storage, таблица `product_images` (поля `kind`,
`color_hex`, `is_primary`, `label`).

> ⚠️ **Важно — двойной кроп:** одно фото показывается в каталоге как **4:5** (плитка) и на
> странице товара как **1:1 (квадрат)**. Снимай с запасом по краям, держи изделие **по центру** —
> чтобы центр-кроп 1:1 не срезал важное. Рекомендуемый мастер: **квадрат ≥ 1500×1500** или
> 4:5 ≥1200×1500 с центрированным изделием.

### 3.1 Основное фото товара (mockup) 📷 снимать
- `kind = 'mockup'`, по возможности **по каждому цвету** (`color_hex`) — тогда при выборе цвета меняется фото.
- Изделие с принтом на нейтральном фоне (крем/светло-серый), мягкий свет.
- Пропорция-мастер: квадрат/4:5, ≥1500×1500. На странице товара рендерится 1:1 с зум-лупой (нужна резкость).
- Минимум: 1 основное фото на товар. Идеал: основное на каждый цвет + 2–3 ракурса.

### 3.2 Lifestyle-фото товара («на человеке») 📷 снимать
- `kind = 'lifestyle'`. Изделие на модели, street-контекст. Та же пропорция-мастер.
- Усиливает страницу товара (отдельная лента миниатюр) и питает галерею «Идеи для старта» на лендинге.

### 3.3 Миниатюры
Генерируются автоматически из тех же фото (64px квадрат) — отдельно готовить не нужно.

# 🟢 БЛОК 4 — Конструктор (база под принт)

### 4.1 Силуэт изделия — сейчас генерится (SVG по цвету) ✅ ассет не нужен
Холст 460×540 рисует силуэт из `garment.ts` (перекрашивается по цвету). Фото не требуется.

### 4.2 Мокап зоны печати (опциональный апгрейд) 📷 снимать / 🎨 рендер
- Поле `print_zones.mockup_url` — реальный мокап изделия, на который накладывается принт
  (полупрозрачно, opacity ~0.55). Если хочешь премиум-реализм вместо генерённого силуэта:
- Фото/рендер пустого изделия **фронт** (и **бэк** для задней зоны) на нейтральном фоне,
  строго фронтально, без перспективы. Пропорция близка к холсту **~46:54** (≈ 0.85:1).
  Размер ≥ 920×1080. По одному на изделие (можно на цвет).
- Это заметно поднимет «вау» конструктора, но не обязательно для запуска.

# 🟢 БЛОК 5 — Принт-библиотека (готовые принты)

### 5.1 Принты библиотеки 🎨 генерировать / 📐 дизайн
- Таблица `print_library` (`file_url` + опц. `thumbnail_url`), грузятся в админке.
- Сами принты: PNG с **прозрачным фоном**, высокое разрешение (для печати — ориентир по
  длинной стороне ≥ 2000–4000 px, 300 DPI на реальный размер зоны).
- Эскиз `thumbnail_url`: квадрат **1:1**, ~600×600, на нейтральном фоне (в модалке показывается `object-cover`).
- Хороший кандидат под AI-генерацию (граффити/стрит-арт/типографика в стиле бренда) + ручная доводка.

# ✅ Приоритеты подготовки

1. **🔴 P0 — бренд/мета:** favicon-набор + OG-default. Без них ссылки и вкладка выглядят «недоделанно». Быстро, дизайн-задача.
2. **🔴 P0 — лендинг:** hero-фото, видео-демо конструктора, 3 макро методов.
3. **🔴 P0 — товары:** хотя бы по 1 качественному фото на активный товар (двойной кроп!).
4. **🟡 P1 — доверие:** 4–6 UGC-фото, lifestyle-кадры товаров.
5. **🟢 P2 — апгрейды:** мокапы зон в конструкторе, принты библиотеки, auth-визуал, фото категорий (когда вернём секцию).

## Что НЕ требует ассетов (генерится/дефолт)
Силуэт изделия в конструкторе (SVG по цвету) · аватары (инициалы) · пустые состояния (иконки) ·
превью дизайнов (скриншот холста) · миниатюры (из фото товара). Блок дизайнеров скрыт фиче-флагом —
слоты `designers.*` не готовим.
