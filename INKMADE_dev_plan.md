# INKMADE — План полной доработки (без оплаты и фискализации)

> Составлен 2026-06-15 по результатам аудита готовности. **Оплата (mock) и фискализация (ОФД РК) НЕ трогаем** — отложены до открытия ИП/договоров. Дизайн и шрифты заблокированы (§0 production_plan) — собираем только из существующих токенов и компонентов.

**Старт:** код MVP ≈ 85%, бизнес-готовность ≈ 55%. Этот план доводит всё, что в нашей власти, до ~95% кода.

---

## Принципы
- Каждый узел доводится до «готово»: `typecheck` + `build` + `vitest` зелёные, smoke-проверка.
- Коммит на каждый завершённый узел (conventional commits).
- Дизайн-токены/шрифты/базовые компоненты — только используем, не переписываем.
- Новые тексты выносим сразу в i18n-ключи (чтобы не переводить дважды).

## БЛОК A — i18n-каркас (ru/kk) · 0% → 70%
**Автономно, не ждёт заказчика (kk-переводы вставим позже).**

- A1. Поставить `@nuxtjs/i18n` (v9), настроить `nuxt.config` (ru default + kk, `prefix_except_default`, cookie-detection).
- A2. Структура `i18n/locales/ru.json` + `kk.json` (зеркало ключей).
- A3. **Проверка казахских глифов** (ә ғ қ ң ө ұ ү һ і) в Unbounded/Manrope. Если Unbounded не держит — fallback-цепочка для kk-заголовков (Golos/Noto), без смены основного шрифта.
- A4. Вынести строки публичного контура в ключи: хедер/футер, лендинг (все секции), каталог, товар, корзина, checkout, заказ/трекинг, login/register, error.vue, тосты `useNotify`.
- A5. Переключатель языка в хедере (desktop + мобильное меню) + футере.
- A6. SEO: `hreflang`, локализованные `useSeoMeta`, `<html lang>` per-locale.
- A7. Числа/валюта/даты через активную локаль (убрать хардкод `ru-RU` из `app/utils/format.ts`).
- **Ждёт заказчика:** живой kk-перевод от носителя (вставим в `kk.json` без изменения кода).

## БЛОК B — Безопасность hardening · 70% → 90%
**Автономно.**

- B1. **Zod на все `readBody`:** `orders/create` (shipping_addr: email/phone/город/адрес), `designs/import` (spec.placements, previewUrl как URL), `promo`, `webhook` (providerTxn).
- B2. **Санитайз загрузки файлов:** magic-bytes (`file-type`), strip `<script>`/`on*` из SVG (или запрет SVG), сузить MIME до png/jpeg/webp, серверный лимит размера.
- B3. **Rate-limit расширить:** `designs/shared/[token]`, `promo/validate`, `designs/import`, попытки логина.
- B4. Перф-БД миграции (Supabase advisor): `auth_rls_initplan` (20 → `(select auth.uid())`), консолидация `multiple_permissive_policies` (115), удалить `unused_index` (12). FK-индексы уже сделаны (0039).
- B5. Auth-политики Supabase: подтверждение email, политика паролей, leaked-password (HIBP), капча на login/register.
- B6. Проверить RPC EXECUTE (13 функций) — по аудиту 13.06 это defense-in-depth, не дыра; зафиксировать решение, не ломать REVOKE.

## БЛОК C — Кабинеты: дизайн-система pass · визуал 65% → 95%
**Автономно.** Привести raw-страницы к `UiAppCard`/`UiAppButton`/`UiSkeleton`/`UiEmptyState` + единые состояния (loading→skeleton, empty→empty-state, error→retry).

- C1. **Admin:** stock, prints, users, audit, pricing, content, legal, settings + карточки заказов/дизайнеров.
- C2. **Studio:** stock, очередь index, карточка заказа.
- C3. **Designer:** finance, prints (форма).
- C4. **Customer:** favorites, addresses, index/профиль.

## БЛОК D — Кабинеты: функциональные дыры · функционал 80% → 95%
**В основном автономно.**

- D1. **Admin:** управление материалами/методами (сейчас только товары); экспорт отчётов по периодам/товарам; расширенный дашборд (тренды). Email-рассылки — опц.
- D2. **Operator:** печать спецификации/этикетки (PDF/QR на упаковку); batch-операции в очереди; экспорт очереди дня (CSV/PDF); уведомление о новых оплаченных заказах.
- D3. **Designer:** аналитика по периодам; история модераций с причинами отклонений; коллекции принтов; соцссылки на витрине; заявка на изменение ставки роялти.
- D4. **Customer:** раздел «Возвраты/рекламации» (статус, подача с фото); инвойс на скачивание; настройки уведомлений (email вкл/выкл); отзывы на товары после доставки.

## БЛОК G — Усиления товара/checkout/маркетинга · 60% → 90%
**Автономно.**

- G1. **Товар:** размерный помощник (рост/обхват → размер из fit_*), размерная таблица в модалке, «похожие изделия» (Embla), отзывы.
- G2. **Checkout:** степпер (контакты→доставка→оплата), обязательный чекбокс согласия, выбор сохранённого адреса, дисклеймер цвета.
- G3. **Маркетинг:** сезонные дропы с таймером (из админки), лента «недавно создано», промо-баннер.

## БЛОК H — Качество: perf / SEO / a11y / тесты · 50% → 90%
**Автономно.**

- H1. `@nuxt/image` на все изображения; explicit width/height; lazy/eager.
- H2. sitemap / robots / динамические OG.
- H3. Lighthouse/CWV (LCP<2.5 / CLS<0.1 / INP<200); reduced-motion уважается.
- H4. a11y-аудит (axe, контраст, клавиатура).
- H5. Sentry (мониторинг ошибок).
- H6. **e2e Playwright** критичного потока: browse→customize→cart→checkout→pay-mock→order.
- H7. Расширить unit-тесты домена; визуальная регрессия ключевых секций.
